## 一、修为值

### 计算公式

```
修为值 = 文档数 × 0.3 + 顶级分类数 × 0.2 + 子分类数 × 0.1 + 标签数 × 0.05
```

| 变量 | 含义 | 修仙寓意 |
|------|------|---------|
| 文档数 | 所有 `.md` 文件 | 吸收天地灵气 |
| 顶级分类 | `docs/` 下直属目录（如 `java/`） | 学习新功法 |
| 子分类 | 顶级目录下的子目录（任意深度） | 修炼新技能 |
| 标签 | 文档 Front Matter 中的 tags | 拓宽见闻 |

### 来源对照

| 行为 | 修仙对应 | 修为影响 |
|------|---------|---------|
| 新增一篇文档 | 吸收灵力 | 修为 +0.3 |
| 新增一个大分类 | 学习更高层功法 | 修为 +0.2 |
| 新增一个小分类 | 学习新技能 | 修为 +0.1 |
| 增加一个标签 | 拓宽见闻 | 修为 +0.05 |
| 更新文档内容（30天内） | 提纯灵力 | 影响灵力速度，不计入境界 |

## 四、灵力速度

灵力速度反映知识库的活跃度，在面板中显示为四种维度的合成值。

| 维度 | 来源 | 修仙含义 |
|------|------|---------|
| 吸收灵力 | 文档总数 | 持续吸入天地灵气 |
| 提纯灵力 | 近30天更新的文档 | 淬炼灵力的纯度 |
| 新技能 | 子分类数 | 修炼各种技能分支 |
| 新功法 | 顶级分类数 | 学习更高层级功法 |

### 速度评级

| 速度值 | 评级 |
|-------|------|
| > 20 | 极速 |
| > 10 | 高速 |
| > 5 | 中速 |
| > 2 | 低速 |
| ≤ 2 | 缓慢 |

## 五、突破与动画

### 触发条件

每次**刷新页面**时，系统会自动对比上次修为与当前修为：

- **小层级精进**：同一大境界内，小层级提升 → 播放「精进」动画
- **大境界突破**：跨越整个大境界 → 播放「突破」动画（含粒子特效 + 光晕扩散 + 光柱）

### 动画效果

突破动画包含：
- 80 颗彩色粒子向四周飘散
- 双层光环向外扩散
- 底部光柱缓缓升起
- 境界文字序列渐入（旧→新）
- 持续 5 秒，点击任意处可提前关闭

### 时间线回放

在修仙面板底部点击「📜 成神之路 · 突破历程」可查看所有历史突破记录：
- 垂直时间轴展示
- 每个节点显示时间、境界变化、修为值
- **点击任一节点可回放当时的升级动画**

## 六、数据存储

突破历程数据跟随 GitHub 账号，存储在 Gist 中：

| 存储方式 | 说明 |
|---------|------|
| Gist（主） | 跨设备同步，跟随 GitHub 账号 |
| localStorage（降级） | Gist 未配置时自动回退 |

> 清理浏览器缓存或在未登录的设备上浏览时，首次记录会从 localStorage 开始；登录 GitHub 并配置 Gist 后自动同步。
